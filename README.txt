TileDesk64 – Tiles Core Pre-Alpha Release

This is the first public Pre-Alpha of TileDesk64, a lightweight tiling desktop environment for the Commodore 64 inspired by i3-style workflows and Amiga-like usability.

The goal of TileDesk is to bring a modern, keyboard-driven, multi-app environment to classic systems while staying fast, minimal, and fully native.

⸻

Current Features (Pre-Alpha)
	•	i3-style tiling window manager
	•	Multiple workspaces (1–4)
	•	Keyboard-driven workflow
	•	Lightweight process system
	•	Integrated applications:
	•	Terminal
	•	Text Editor (Pro Mode)
	•	Commander (file navigation)
	•	System Monitor (htop-style)
	•	GPT stub (future AI integration)
	•	SID player (basic)
	•	Overlay menu system (non-destructive UI)
	•	Dirty-render optimization (fast redraws on C64)
	•	File system stub with basic operations

⸻

Controls

General:
	•	RUN/STOP → Enter command mode
	•	Cursor Keys → Navigate focus between tiles
	•	1–4 → Switch workspace

Command Mode (after RUN/STOP):
	•	m → Open app menu
	•	v → Next split vertical
	•	h → Next split horizontal
	•	c → Spawn Commander
	•	e → Spawn Editor
	•	t → Spawn Terminal
	•	p → Spawn System Monitor
	•	g → Spawn GPT
	•	s → Spawn SID player
	•	x → Kill focused tile

Menu:
	•	Cursor Keys → Navigate
	•	ENTER → Execute
	•	RUN/STOP → Close menu

⸻

Editor Controls (Pro Mode)
	•	Type normally to insert text
	•	ENTER → New line
	•	Backspace → Delete before cursor
	•	DEL → Delete at cursor
	•	Cursor Left/Right → Move cursor
	•	Cursor Up/Down → Move between lines
	•	CTRL+A → Move to line start
	•	CTRL+E → Move to line end

Features:
	•	Multi-line editing
	•	Scrolling
	•	Line numbers (toggle via menu)
	•	Preview mode (toggle via menu)
	•	Insert mode (no overwrite)

Vision

TileDesk aims to unify:
	•	Commodore 64
	•	Amiga systems
	•	i386 / x64 platforms

into a shared tile-based UI concept where applications can run in a consistent environment across architectures while maintaining compatibility with:
	•	BASIC workflows
	•	Classic Amiga software paradigms
